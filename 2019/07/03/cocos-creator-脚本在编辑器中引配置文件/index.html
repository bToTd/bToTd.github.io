cocos creator 脚本在编辑器中引配置文件

发表于 2019-07-03

cocos creator 的脚本分为两种运行环境，一个编辑器中运行，另一个就是在游戏正在跑起来的运行（浏览器，模拟器）。在浏览器中运行都好说，什么变量赋值，参数传递，监听事件什么的，都可以处理的很好（处理内存管理，要受限制与JavaScript语言的限制，不能自己管理）。

脚本在creator中运行时，参数传递的就会出一些问题。其实也不能全算creator的问题，只是在一些写好的工具，框架中。唉，要打成一个 js 给别人用。这时候就不能使用 import 导入外部的配置文件，这就很扯淡。我靠不开源的框架怎么能用。然后呢，使用其他办法。

creator中Editor这个类里提供了一个Editor.require()方法，可以导入外部的配置文件，在没有打成一个 js 文件时，一切都是好的。无论是ts，js，json的文件，都可导进来。但是，在打成一个 js 文件之后，导 ts 文件就出错了。导json的却是可以的。

只能将就的导 json 文件了。
