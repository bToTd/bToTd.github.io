nodejs调用shell

发表于 2019-07-01

node 调用shell命令，可以自动的更新一些文件，不如自动拉去同步git项目这些操作。还有很多其他逼格高的场景。

官方文档: http://nodejs.cn/api/child_process.html

node child_process 模块

child_process 这个模块是node调用shell命令或者执行shell脚本文件的模块。无论是nodejs还是python，都是直接开一个子进程，然后执行。

这个模块中我用的比较多的接口是child_process.exec(command[, options][, callback])。这个接口是一步开启子线程。官方给出的例子比较简单明了，很容易理解。

上面的例子只用运行的命令和回调函数，并没有设置参数，其中options这个对象中可以包含一下参数：
options

  • cwd 子进程的当前工作目录（shell命令在哪个路径下运行）。默认值: null。
  • env
  • encoding 默认值: ‘utf8’。
  • shell 用于执行命令的 shell。参阅 shell 的要求与 Windows 默认的 shell。 默认值: UNIX 上是 ‘/bin/sh’，Windows 上是 process.env.ComSpec。
  • timeout 默认值: 0。
  • maxBuffer stdout 或 stderr 上允许的最大字节数。如果超过限制，则子进程会被终止并且截断任何输出。参阅 maxBuffer 与 Unicode 中的警告。默认值: 1024 * 1024。
  • killSignal | 默认值: ‘SIGTERM’。
  • uid 设置进程的用户标识，参阅 setuid(2)。
  • gid 设置进程的群组标识，参阅 setgid(2)。
  • windowsHide 隐藏子进程的控制台窗口（在 Windows 系统上通常会创建）。）。默认值: false。
