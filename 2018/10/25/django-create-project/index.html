文件说明

django生成项目和应用时，都会自动生成一些文件，下面就来说一下这些文件的作用。

manage.py 和 django-admin.py

创建项目时，我们用到的是django-admin.py，这是 Django 的任务管理命令行工具。可以通过django-admin
来查看帮助信息和所支持的命令行参数。

在创建一个Django项目后，manage.py被自动生成在项目的根目录下，它是对django-admin.py的简单封装。
查看一下这个的命令行参数：

manage.py和djanggo-admin有相同也有不同，这写区别要了解可以读官方文档。官方文档是最初始的
和最详细的说明。官方文档链接：https://docs.djangoproject.com/en/1.10/ref/django-admin/

特别注意：项目根目录下的manage.py不要修改，不要删除。

mysite 项目名目录

mysite是所建目录的管理功能目录，下面是简单介绍一下该目录下的各个文件的功能。

  1. settings.py 这个文件包括了项目的初始化设置，可以针对整个项目进行有关参数配置，比如配置数据
    库，添加应用等。
  2. urls.py 这是一个URL配置表文件，主要是将URL映射到应用程序上。当用户请求某个URL时，Django项目会根据
    某个应用中的urls.py文件，也可以是某个具体的视图函数。在Django中，这个文件也被称为URLconf，这是
    Django非常强大的一个特性。
  3. wsgi.py WSGI 是Web Server Gateway Interface 的缩写。WSGI是python所选择的服务器和应用标准，
  4. __pycache__：这个在网站运行后才出现的，它就是一个编译厚的文件夹。

blog 应用目录

blog 是在项目中所创建的应用之一，在项目中每创建一个应用就会生成相应的文件目录。这些文件目录下
都会自动的生成一些文件，这些文件的功能：

  1. admin.py 在这个文件中，可以自定义Django管理工具，比如设置在管理界面能够管理的项目，或者通过
    重新自定义与系统管理有关的类对象，向管理功能增加新的内容。
  2. apps.py 这个文件是Django1.10之后增加的，包含对应用的配置，比如为管理功能提供一个合适的应用名字。
  3. migrations 这是一个目录，用来存储应用的数据库表结构的指令，通过这些指令可以修改和创建数据库，，
    从而在modles.py模型类中和数据库表之间迁移。
  4. models.py 这是应用的数据模型，每一个Django应用都应当有一个models.py，虽然该文件可以为空，但是
    最好不要删除。
  5. test.py 这个文件中可以编写测试文档来测试所建立的应用。
  6. views.py 这是一个重要的文件，用户保存响应各种请求的函数或者类。如果编写的是函数，则称为基于
    函数的视图; 如果编写的是类，则称为基于类的视图。views.py就是保存函数或者类的视图文件。当然，
    也可以用其他的文件名称，只不过在引入相应函数或者类时，要注意名称的正确性，views.py是习惯命名。

db.sqlite3

这是一个默认的数据。SQLite 是 python 默认安装的数据库，在Django中也是默认使用的。

网站的配置

前面仅仅创建了应用，还没有将该应用注册到本项目中，因此暂时还不能访问。
在Django项目中，主管信息信息注册的文件是./mysite/settings.py。

  1. DEBUG 开发是设为True

  2. INSTALLED_APPS 注册应用，你要将你新建的应用加入到这个列表里。

  1. ALLOWED_HOSTS 域名列表，设置通过域名访问。

  2. 配置数据库，默认是QSList，我在这也没有改，我的小霸王每次开MySQL都会很卡。
