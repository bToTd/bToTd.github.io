vhdl 代码结构

发表于 2018-10-08 | 分类于 数字电路设计

一段完整的 vhdl 代码有 3 个基本组成部分：库声明(library), 实体(entity)和构造体(architecture),
由于 vhdl 大小写不敏感，所以之后的代码都用小写了。

vhdl 代码基本单元

一段独立的 vhdl 代码至少包含 3 个组成部分：

  • 库(library)声明：列出了当前设计中需要用到的所有库文件，如 ieee，std，和 work 等。
  • 实体(entity)：定义了电路的输入和输出引脚。
  • 构造体(architecture)：所包含的代码描述了电路要实现的功能。

库是一些常用代码的集合，将电路设计中重复使用的代码存放到库中，方便之后的调用。代码通常以函数
(function), 过程(procedure), 或者元件(component) 等标准形式存放在包集(package)中，用户可以根据
需要对其进行编译使用。

库的声明

库(library) 的建立和使用有利于设计重用和代码共享，同时可以使代码结构更加清晰。

库的种类

在 vhdl 设计中有 3 种常用的库：ieee 库， std 库和 work 库。

ieee 库

在 ieee 库中有一个 IEEE 正式认可的标准包集 std_logic_1164.all。实际上，ieee库中包含了许多包集，列举
如下：

  • std_logic_1164: 定义了 STD_LOGIC(8值) 和 STD_ULOGIC (9值) 多值逻辑系统。
  • std_logic_arith: 定义了SIGNEND (有符号) 和 UNSIGNEND (无符号) 数据类型和相关的算术运算和比较运算操
    作。它包含许多数据类型转换函数，这种函数可以实现数据类型的转换。常用的数据类型转换函数包括
    conv_integer(p), conv_unsigned(p, b), conv_signed(p, b), conv_std_logic_vector(p, b)
  • std_logic_signed: 内部包含一些函数，这些函数可以使 STD_LOGIC_VECTOR 类型的数据像 SIGNED 类型的数据
    一样进行运算操作。
  • std_logic_unsigned: 内部包含一些函数，这些函数可以使 STD_LOGIC_VECTOR 类型的数据像 UNSIGNED 类型的
    数据一样进行运算操作。

std 库

std 库是 vhdl 设计环境的标准资源库，包括数据类型和输入输出文本等内容。std 库中存放有包集 standard 和
textio。

work 库

work 库是当前工作库，当前设计的所有代码都存放在 work 库中，使用 work 库不需要进行声明。

库的声明

使用一个库之前，要先对库进行声明。经过声明之后，在设计中就可以调用库中的代码。库的声明分为两个步：

  1. library 库的名字；
  2. use 库的包集；
